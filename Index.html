    Introduction

    Welcome to the Recording API! You can use our API to access Recording API endpoints, which can get information on various Recording files, and folders in our database for your account.

    We have language bindings in Shell, C#, JAVA, PHP, GO and many more! You can view code examples in the dark area to the right, and you can switch the programming language of the examples with the tabs in the top right.

    Authentication

    Make sure to replace 553a431e192d2553a431e1930f with your API key.

    Recording uses API keys to allow access to the API. You can register a new Recording API key at the time of verification.

    Recording expects for the API key to be included in all API requests to the server in a header that looks like the following:

    api_key: 553a431e192d2553a431e1930f

    You must replace 553a431e192d2553a431e1930f with your personal API key.

    Register

    You need to register on the app using your phone number with country code to get verified the phone and get API key

    Register Phone

    The Register Request Code Example: 

    
curl -X POST https://app2.virtualbrix.net/rapi/register_phone/ -d \
"phone=+16463742122&
token=55942ee3894f51000530894"
    
using Recording;
Api.Token="55942ee3894f51000530894";  
var resp = Api.RegisterPhone("+1", "3133073896");

    The above command returns JSON structured like this:

    
{
    "status":"ok",
    "phone":"+16463742122",
    "code":"54004",
    "msg":"Verification Code Sent"
}

    This endpoint register your phone number with the app.

    curl Request

    https://app2.virtualbrix.net/rapi/register_phone/

    Parameters

    Parameter Required Comments
    phone Yes the phone number with country code
    token Yes the value of the token

    Verify Phone

    The Verify Phone Request Code Example: 

    
curl -X POST https://app2.virtualbrix.net/rapi/verify_phone/ -d \
"phone=%2B16463742122
&code=54004&mcc=300
&app=free&token=55942ee3894f51000530894
&device_token=234
&device_type=android
&time_zone=10"
    
using Recording;   
VerifyPhoneRequest verifyPhoneReq = new VerifyPhoneRequest
{
    Token = "55942ee3894f51000530894";,
    Phone = "+16463742122",
    App = "rec",
    Code = "96080",
    DeviceId = "yourDeviceId",
    DeviceToken ="yourdevicetoken",
    DeviceType = DeviceType.windows.ToString(),
    Mcc = "300",
    TimeZone = "330"
};
var status =  Api.VerifyPhone(verifyPhoneReq);

    The above command returns JSON structured like this:

    
{
    "status":"ok",
    "phone":"+16463742122",
    "api_key":"557872b508520557872b50855c",
    "msg":"Phone Verified",
    "code":"phone_verified",
}

    This endpoint verify your phone number with the app.

    curl Request

    https://app2.virtualbrix.net/rapi/verify_phone/

    Parameters

    Parameter Required Comments
    phone Yes the phone number with country code
    code Yes the verification code reciever in the previous call
    token Yes The token set from server
    mcc Yes the integer value e.g. "300"
    app Yes the value of the app values are free/rec/rem
    device_token Yes the device token that can be generated from device
    device_type Yes the device type e.g. android or iOS
    time_zone Yes the time zone difference in minutes

    Create File

    The Create File Request Code Example:

    
curl -X POST https://app2.virtualbrix.net/rapi/create_file/ -F
"file=@/var/www/apps/phone/message.wav" -F
"api_key=557872b508520557872b50855c" -F

"name":"testfile",
"notes":"testnotes",
"remind_days":"10",

using System.IO;
using Recording.Classes;
using Recording;
Api.ApiKey = "557872b508520557872b50855c";
var strRecFile = "C:\\audioFile.mp3";
var recFileName =Path.GetFileName(strRecFile);
HttpFile fileParam = new HttpFile();
fileParam.Name = recFileName;
fileParam.FileName = recFileName;
FileStream fstream = new FileStream(strRecFile, FileMode.Open, FileAccess.Read);
byte[] fileData = new byte[fstream.Length];
var dataLength = fileData.Length;

fstream.Read(fileData, 0, fileData.Length);
fstream.Close();
fileParam.ContentType = "application/octet-stream";
CreateFileRequest createRecFile = new CreateFileRequest
{
    Notes = "Test Notes",
    Tags = "Files Tags",
    Name = recFileName,
    RemindDays = 5,
    RemindDate = DateTime.Now.AddDays(5)
};

Api.CreateFile(fileData, createRecFile, fileParam);

    The above command returns JSON structured like this:

    
{
    "status":"ok",
    "msg":"File Uploaded Successfully",
    "id":"1"
}

    Errors

    This error section listed.

    The Recording API uses the following error codes:

    Error Code Meaning
    400 Bad Request -- Your request is invalid.
    401 Unauthorized -- Your API key is wrong.
    403 Forbidden -- The file requested is hidden for administrators only.
    404 Not Found -- The specified file could not be found.
    405 Method Not Allowed -- You tried to access a file with an invalid method.
    406 Not Acceptable -- You requested a format that isn't json.
    410 Gone -- The file requested has been removed from our servers.
    418 I'm a teapot.
    429 Too Many Requests -- You're requesting too many files! Slow down!
    500 Internal Server Error -- We had a problem with our server. Try again later.
    503 Service Unavailable -- We're temporarily offline for maintenance. Please try again later.
